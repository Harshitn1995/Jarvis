Welcome to Jarvis's guide to getting started with Users.

Who is a User?

At Jarvis, anybody who has interacted with your business at least once is called a User.

This includes visitors who engage via your mobile app, website, and even offline channels.

How does Jarvis detect users?

By connecting your platforms (app, website, CRM), Jarvis auto-detects users in real time.

  • Users: Basic information like location, browser, OS
  • Events: Activities like login, logout, app installs

ðŸ“Œ Unique Identifiers

Details like email, phone number, or customer ID are called Unique Identifiers in Jarvis.

Tip: Use Identifying Users section for setup instructions.

User Attributes

They define user preferences, personal details, and behaviors:

  • System Attributes: Auto-collected (e.g. DOB, Gender)
  • Custom Attributes: Business-defined (e.g. Reward Points, Membership Tier)

User Profiles

Every user profile includes data points across five major categories:

  • Basic Info: Contact, Activity, Source, Campaigns
  • Attributes: Personal and Behavioral Data
  • Devices: Device type, session stats, browser info
  • Channels: Email, Push, SMS, etc.
  • Events: Logged user actions like checkout, add to cart

ðŸ“Œ Note on Events

Every action users perform on your platform becomes an event (e.g. view product, complete purchase).
